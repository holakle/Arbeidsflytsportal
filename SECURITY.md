# Security Policy

## Reporting a Vulnerability

Please report security vulnerabilities through one of these channels:

- Preferred: GitHub Security Advisories (private report in this repository)
- Fallback: contact repository maintainers directly

If you are unsure how to contact maintainers, open a normal issue **without** sensitive details and ask for a private channel.

Do **not** post secrets, tokens, private keys, passwords, or exploit details in public issues.

## Expected Response Time

- Initial triage response: within 3 business days
- Status update after triage: within 7 business days
- Target for remediation plan: within 14 business days (severity dependent)

## Supported Versions

Security fixes are prioritized for:

- `main` branch (latest)
- Most recent release branch, if maintained

Older, unmaintained branches may not receive security patches.
